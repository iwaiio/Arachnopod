# ДОКУМЕНТАЦИЯ

## ЦЕЛЬ
    Главной задачей проекта является создание восьминогого робота для получения опыта разработки и тестирования в области робототехники, компьютерного зрения, программирования и электротехники.

## СТЕК
    Используется:
    • Языки программирования: C/C++; Python.
    • Целевая платформа CS: Orange Pi 5; OS: Armbian (Linux).
    • ПК для разработки/тестирования: Windows.
    • MCU: STM32.
    • Сборка:
      • CS/симуляция/утилиты: CMake.
      • Прошивки STM32: STM32CubeIDE (toolchain arm-none-eabi).

    Планируется / в разработке:
    • Middleware: ROS 2 (интеграция компонентов CS и симуляции).
    • Computer Vision: OpenCV (обработка); GStreamer (видеопайплайн/стриминг).
    • Сетевое взаимодействие с сервером: MQTT и/или REST API.

## ТЕРМИНЫ И СОКРАЩЕНИЯ
    Система - самостоятельный узел, имеющий MCU/CPU и собственное ПО, является объектом управления и оконечным устройством на шине.
    Модуль - периферийный узел, не является объектом управления.
    Программный модуль - логический компонент ПО внутри системы (например, внутри CS или MNS).

    ПО - программное обеспечение
    ПМ - программный модуль

    CS  - Computing System (Вычислительная система)
    PSS - Power Supply System (Система электроснабжения)
    TCS - Telemetry Collection System (Система сбора информации)
    TMS - Thermal Management System (Система обеспечения температурного режима)
    MNS - Mobility and Navigation System (Система движения и навигации)
    INM - Inertial Navigation Module (модуль инерциальной навигации)
    DCM - Drive Control Module (модуль приводов)
    NM  - Network Module (Сетевой модуль)
    CVM - Computer Vision Module (Видео модуль)
    AM  - Audio Module (Аудио модуль)
    LS  - Lighting System (Светотехническая система)

## ВВЕДЕНИЕ
    Проект представляет собой модульную робототехническую платформу, построенную по централизованной архитектуре управления. В роли головного вычислительного узла выступает Computing System (CS) на базе Orange Pi 5 под управлением Linux (Armbian). CS выполняет функции высокоуровневой логики: обработку телеметрии, принятие решений, управление остальными системами, а также сетевое взаимодействие с внешним сервером по Wi-Fi.

    Обмен между системами построен по централизованной схеме: CS является инициатором обмена по шине и источником управляющих команд, а остальные узлы отвечают на запросы CS в рамках заданного протокола. Для снижения нагрузки на шину первичные сигналы датчиков и статусы агрегируются и буферизуются в Telemetry Collection System (TCS), после чего CS выполняет их опрос и получает телеметрию без прямых обращений к датчикам. Система движения и навигации Mobility and Navigation System (MNS) реализует полный контур управления движением: расчёт кинематики конечностей, ПИД-регуляторы приводов, диагностику и локальные защиты. MNS использует следующие модули: Inertial Navigation Module (INM) формирует оценку ориентации и динамики по IMU и барометру для стабилизации и коррекции движения; Drive Control Module (DCM) управляет приводами и выполняет низкоуровневые контуры. Факт упора конечности определяется в MNS по измерениям тока приводов. Питание систем и модулей обеспечивается Power Supply System (PSS), температурный режим поддерживает Thermal Management System (TMS). Модули CVM/AM/LS обеспечивают восприятие и взаимодействие.

## СОСТАВ
    Аппарат состоит из нескольких систем и модулей:

    1. Вычислительная система (Computing System) - Orange Pi 5, NM, CVM, AM.
        Целевые задачи:
        1.1. Связь с внешним сервером, выход в сеть по Wi-Fi;
        1.2. Обработка видеоинформации и работа алгоритмов компьютерного зрения, составление 3D-карты местности;
        1.3. Управление остальными системами аппарата;
        1.4. Принятие решений и работа логики высокого уровня;
        1.5. Обработка телеметрии;
        1.6. Работа LLM.

    2. Система электроснабжения (Power Supply System) - Li-ion батарея 6S2P, BMS, преобразователи напряжения.
        Целевые задачи: контроль и управление питанием всех систем аппарата.
    
    3. Система сбора информации (Telemetry Collection System) - STM32.
        Целевые задачи: агрегирует первичные сигналы датчиков и систем, буферизует и передаёт их в CS, исключает прямую нагрузку на CS.
    
    4. Система обеспечения температурного режима (Thermal Management System) - STM32, PWM-контроллер, кулеры, термодатчики.
        Целевые задачи: регулирование температуры греющихся элементов, управление кулерами, отвод тепла.

    5. Система движения и навигации (Mobility and Navigation System) — STM32, INM, DCM.
        Целевые задачи:
        5.1. Управление манипуляторами робота, перемещение в пространстве;
        5.2. Расчёт кинематики конечностей и генерация движений и походки;
        5.3. ПИД-регуляция приводов и формирование управляющих воздействий;
        5.4. Контроль корректности движения, локальные защиты и диагностика;
        5.5. Определение упора конечности по току приводов;
        5.6. Навигация.

    6. Модуль инерциальной навигации (Inertial Navigation Module) — IMU, барометр.
        Целевые задачи: оценка ориентации и ускорений по IMU и измерение высоты по барометру, обработка первичных данных.

    7. Модуль приводов (Drive Control Module) — PWM-контроллеры, сервоприводы.
        Целевые задачи: управление приводами.

    8. Сетевой модуль (Network Module) - Wi-Fi модуль, антенны.
        Целевые задачи: прием-передача данных по Wi-Fi и Bluetooth.

    9. Видео модуль (Computer Vision Module) - бинокулярная камера.
        Целевые задачи: передача видеоинформации в CS с последующей обработкой.

    10. Аудио модуль (Audio Module) - динамики, микрофон.
        Целевые задачи: распознавание и синтез голоса, взаимодействие с пользователем.

    11. Светотехническая система (Lighting System) - STM32, набор светодиодов.
        Целевые задачи: индикация состояния систем, освещение темных участков для корректной работы CVM, косметические эффекты.

## ПРОТОКОЛ ОБМЕНА ДАННЫМИ
### Общие сведения
    CS - контроллер шины. Оконечные устройства могут только отвечать на сообщения от контроллера шины.

    Флаги сообщения:
        0. CS высылает оконечному устройству управляющее воздействие.
        1. Оконечное устройство отвечает на управляющее воздействие и высылает ответное сообщение.
        2. CS высылает оконечному устройству запрос на передачу данных.
        3. Оконечное устройство отвечает на запрос данных и высылает ответные данные.

    Сообщения состоят из нескольких частей: "начальный байт", "заголовок", "данные", "конечный байт". 

    - Начальный байт 0xAA (8 бит).
    - Заголовок содержит: бит четности (1 бит); адрес оконечного устройства (5 бит); флаг сообщения (2 бита); дополнительный маркер (3 бита); N блоков данных (5 бит). На заголовок выделяется 16 бит.
    - Данные состоят из N блоков по 16 бит. В каждом блоке данных содержится: данные (16 бит).
    - Конечный байт 0x55 (8 бит).

    Дополнительный маркер - резерв.

    CS ожидает ответное сообщение в течение 25 мс после инициации обмена данных. По истечении ожидания (25 мс) обрабатывается сценарий ошибки "срыв обмена данных".

### Описание
    Бит четности вычисляется путем подсчета количества единиц в двоичном коде сообщения. При четном количестве едениц бит четности равен нулю, иначе единице.

## Вычислительная система CS
    CS является Orange Pi 5 с операционной системой Linux Armbian.
    Стек программного обеспечения OP5:
        • CS_BUS_CONTROL — мастер-шина, планирование опроса систем, отправка команд.
        • CS_OPERATING_MOD — управление режимами работы аппарата, основная логика (точка входа main).
        • CS_BASE — хранение конфигурации/логов/состояния (механизм хранения уточняется).
        • CS_PSS — управление/мониторинг PSS.
        • CS_TCS — управление/мониторинг TCS.
        • CS_MNS — управление/мониторинг MNS (выдача команд движения/траекторий).
        • CS_TMS — управление/мониторинг TMS.
        • CS_LS — управление LS.
        • CS_NM — сетевое взаимодействие.
        • CS_CVM — обработка данных CV (в разработке).
        • CS_AM — обработка аудио (в разработке).
        • CS_LLM — сопряжение с LLM.
    OP5 напрямую связан с модулями: NM; CVM; AM.
    OP5 управляет следующими системами, подключенными к шине данных: PSS; TCS; MNS; TMS; LS.

## РЕЖИМЫ РАБОТЫ 
    1. Зарядка;
    2. Энергосберегающий режим;
    3. Активное исследование, составление 3D-карты местности;
    4. Режим управления LLM;

## ОГРАНИЧЕНИЯ И КОМПРОМИСЫ
    Вычислительные ограничения: учитываются возможности OP5 и STM32. Системы, не требующие сложных вычислений, обслуживаются на STM32; ресурсоёмкие задачи (CV/высокоуровневая логика) выполняются на OP5.
    Ограничения питания: PSS обеспечивает питание всех систем; целевые значения пиковой мощности и времени автономной работы уточняются расчётом энергобаланса. Для снижения перегрева предусмотрена терморегуляция (TMS).

## РАЗРАБОТКА ПО
### Архитектура проекта
    *** Префикс i_ в названии файлов обозначает, что файл содержит интерфейсные функции.
    Пример управления PSS через CS: CS отрабатывает алгоритм ПО управления PSS, который использует интерфейсные функции для взаимодействия с PSS, которые описаны в i_pss.hpp. Интерфейсные функции дают возможность считывать приходящие данные из PSS и формировать УВ в PSS. CS отправляет сформированные с помощью интерфейсных функций сообщения, которые расшифровываются в PSS. далее PSS выполняет выданные команды и изменяет физическое состояние системы. в ПО тестирования физическое состояние системы изменяется с помощью использования интерфейсных функций i_pss_sim.hpp.

    cs.cpp <- cs.hpp <- cs_systems.hpp <- cs_pss.cpp <- i_pss.hpp <- pss.cpp <- pss.hpp <- i_pss_sim.hpp <- pss_sim.c <- pss.hpp
